SendMessage API

SendMessage API'yi kullanarak kullanıcılarınıza metin, resim, görüntü, video, ses, dosya ve yapılandırılmış mesaj şablonları üzerinden mesajlar iletebilirsiniz.

İstek URI'ı#

https://<DIYALOG-SERVER-API-ENDPOINT>/v1/bots/v1/bots/sendmessage/<YOUR-BOT-ACCESS-TOKEN>

DIYALOG-SERVER-API-ENDPOINT : DLG sunucusun api endpoint adresidir. Örnek olarak api.dlgmessenger.com verebiliriz.

YOUR-BOT-ACCESS-TOKEN : DLG Platformunda uygulamalar üzerinden kullanıcılarınıza tüm mesajları sizin için Platformun yöneticisi tarafından oluşturulan bir bot kullanıcısı üzerinden gönderebilirsiniz. İsterseniz tüm uygulamalarınızdan sadece bir bot üzerinden mesajlar gönderebilir veya uygulamalarınızın fonksiyonelitesine göre farklı botlar yaratıp, her uygulamadan farklı bot aracılığı ile mesaj gönderebilirsiniz. Yaratılan Bot, kullanıcılarınızın mobil cihazında bir kişi gibi görünür. Bu kişi aracılığı ile uygulamanız ve müşteriniz arasında mesajlaşma yapabilir durumda olursunuz.
Her oluşturulan bot'un bir token'ı vardır. Bu token'ı DLG yöneticinizden almanız ve URI'ın sonuna eklemeniz gerekmektedir. DLG sunucusu mesajı iletirken token'ı algılar ve hangi bot üzerinden kullanıcıya mesaj göndereceğini bulur ve kullanıcıya mesajı iletir.
Bu token gizlidir. Eğer yetkisi olmayan biri tarafından bu token alınırsa sizin bot'unuz üzerinden kendi mesajlarını iletebilir. Bu nedenle güvenliğinin sağlanması gerekmektedir.

Örnek İstek#

Aşağıda komut satırından curl ile çalıştırabileceğiniz örnek bir http isteğini görebilirsiniz.

curl -X POST -H "Content-Type: application/json" -d '{
"recipient": {
"appCustomerId":"1231231"
},
"message":{
"text":"hello world",
"messageId": "msg-id-123-112233"
"quick_replies":[]
}
}' "https://api.dlgmessenger.com/v1/bots/sendmessage/c2dc9bf3e0c3218e9ab22b56002e4644"

İstekte de görülebileceği üzere istek URI'i https://api.dlgmessenger.com/v1/bots/sendmessage/c2dc9bf3e0c3218e9ab22b56002e4644 olarak verilmiştir. Bu uri'a yine yukarıdaki istekte görüleceği üzere JSON objesi POST metodu ile gönderilir.

Bu endpoint adresi parçalarsak;
Api Server : api.dlgmessenger.com
Api Adı : /v1/bots/sendmessage/
Bot token : c2dc9bf3e0c3218e9ab22b56002e4644

İstek Json Objesi Detayları#

Api isteği iki alandan oluşmaktadır. Bu alanla birer objedir ve kendi içinde değişen alanlar alabilmektedir. recipient objesi mesajın alıcısının api'ye iletildiği alan, message objesi ise mesajın içeriğinin verildiği alandır.

AlanTipAçıklamaZorunlu Alan
recipientObjectAlıcı objesi (recipient objesi)Evet
messageObjectiçerik objesi (message objesi)Evet

SendMessage API'sine yukarıdaki iki alanı içeren JSON objesi POST metodu ile gönderilmelidir.

{
"recipient":{
...
},
"message":{
...
}
}

recipient#

Mesajı iletmek istediğiniz kullanıcı bilgisini, recipient objesi içerisinde api'ye iletebilirsiniz.

PropertyTipAçıklamaZorunlu Alan
appCustomerIdStringMüşteri numarasıdır.Opsiyonel
appStaffIdStringÇalışan sicil numarasıdır.Opsiyonel
idStringDLG tekil kullanıcı numarasıdır.Opsiyonel

Mesajı hangi kullanıcıya göndermek istediğinizi bu alan ile api'ye iletirsiniz. İstek appCustomerId, appStaffId veya id öğelerinden sadece birini içermelidir.

DLG sisteminde kullanıcıların iki tekil id'si vardır. Bunlardan biri DLG kullanıcı numarasıdır. Bu numara gelişigüzel oluşturulur ve her kullanıcı için sabit ve tekildir. Diğer id ise platformu kullanan kurumların kendi bildikleri müşteri numarası veya sicil numarasıdır. DLG sisteminde bir kullanıcı yaratıldığında DLG id'si ve müşteri numarası veya kurumun çalışanları için sicil numarası ile kullanıcı yaratılır. Bu nedenle bir kullanıcınıza mesaj iletmek için bu üç id'den birini kullanabilirsiniz.

Müşterilerinize mesaj iletmek istiyorsanız appCustomerId alanını yani müşteri numarasını, kurum çalışanlarınıza mesaj iletmek istiyorsanız appStaffId alanını yani sicil numarasını kullanmalısınız. DLG id'si bu iki seçeneğe alternatiftir. Eğer kullanıcınızın DLG id'sini biliyorsanız direk bu id'yi kullanabilirsiniz.

Uygulamalarınız üzerinden mesaj gönderirken en doğrusu DLG id'sini (id) kullanmak yerine kendi bildiğiniz id'yi yani müşteri numarasını (appCustomerId) veya sicil numarasını (appStaffId) kullanmaktır. Bu sayede DLG özelinde bir veriye erişim ihtiyacınız olmayacaktır.

Örnek recipient json objesi#
{
"recipient": {
"appCustomerId": "123456789"
},
"message": {
...
}
}

message#

Kullanıcılarınıza iletmek istediğiniz mesajların içeriğini bu alan içerisinde api'ye iletebilirsiniz.

message objesi text, attachment, quickReplies, quickRepliesV2 temel öğelerinden oluşmaktadır. text veya attachment alanlarından aynı anda **sadece birini** kullanabilirsiniz.text alanını kullandığınızda bu sizin kullanıcıya sadece metin mesaj göndermek istediğiniz anlamına gelir. text yerine attachment kullanırsanız bu kullanıcıya medya, dosya veya hazır şablon mesajlardan birini göndermek istediğiniz anlamına gelir.

Metin mesajlarınıza "evet"-"hayır" gibi hızlı cevap düğmeleri ekleyebilirsiniz. Bunun için quickRepliesV2 alanını kullanabilirisiniz.

Örnek message json objesi#
{
"recipient": {
...
},
"message": {
"messageId" : "msg-id-123-1122-1111",
"text" :"hello world ¯\_(ツ)_/¯",
"quickReplies" : [],
"quickRepliesV2" : []
}
}
PropertyTipAçıklamaZorunlu Alan
messageIdStringSisteminizde oluşturduğunuz tekil mesaj id'sidir. Tüm mesaj iletimleri ve cevapları vs. gibi eventleri DLG bu id ile size besler. Eğer id'yi vermezseniz DLG sizin için bir id üretecek ve bu id üzerinden eventleri iletecektir.Opsiyonel
textStringEğer kullanıcıya yalnızca metin mesaj göndermek isterseniz bu alanı apiye iletmeniz gerekiyor. Metin yerine diğer türlerde mesaj iletmek istiyorsanız text alanı yerine attachment alanı apiye vermelisiniz.Opsiyonel
attachmentObject(attachment Objesi) Metin dışındaki diğer türlerdeki mesajların içeriklerini bu alanı api'ye iletebilirsiniz.Opsiyonel
quickRepliesArray<quickReply>Bu alan eski versiyondur fakat zorunlu olduğu için tüm isteklere "quickReplies" : [] olarak eklenmelidir. İçeriği boş olarak verilmelidir.Evet
quickRepliesV2Array<quickReplyV2>quickReplies parametresinin 2. versiyonudur. Eğer metin mesaja hızlı cevap butonları eklemek için bu alanı kullanabilirsiniz.Evet
Not :

quickReplies, quickRepliesV2'nin eski versiyonudur. Yeni versiyon mobil istemciler quickRepliesV2'yi kullanmaktadır. Fakat yinede eski versiyonu "quickReplies" : [] olacak şekilde her isteğe eklenmelidir. Ayrıca buton eklenmese bile "quickRepliesV2" : []" elementi isteğe eklenmelidir. Aksi halde sunucudan malformed request error hatası alırsınız.

Aşağıda metin mesaj gönderimi için örnek bir json objesi görebilirsiniz. Bu istekte 123456789 müşteri numaralı kullanıcıya hello world ¯_(ツ)_/¯ mesajı iletilmektedir.

{
"recipient": {
"appCustomerId": "123456789"
},
"message": {
"messageId" : "msg-id-123-1122-1111",
"text" :"hello world ¯\_(ツ)_/¯",
"quickReplies" : [],
"quickRepliesV2" : []
}
}

attachment#

Eğer metin dışında aşağıda diğer türlerde mesaj göndermek istiyorsanız text elementi yerine attachment alanını kullanmalısınız.

attachment elementi kullanarak kullanıcılarınıza aşağıdaki türlerde mesaj iletebilirsiniz.
  • Medya mesajlar : resim, ses, video veya dosya.
  • Hazır şablon mesajlar : generic template, button template
PropertyTipAçıklamaZorunlu Alan
typeStringattachment türleri: image, audio, video, file veya templateEvet
payloadObjectiletilen ekin içeriğidir. payload objesi.Evet

Dönen Yanıt (Response)#

Başarılı sonuçlanan API istekleri http-2XX cevap kodu ile json objesini döner. Bu cevap içerisinde mesajın iletildiği kullanıcının DLG tekil id'si ve mesaj id'si döner. Eğer istek içerisinde bir mesaj id verilmiş ise aynı mesaj id size geri dönecek eğer verilmemiş ise DLG tekil bir mesaj id oluşturup size iletecektir.

Yanıt Json Objesi Detayları#

PropertyTipAçıklama
recipient_idStringTekil DLG kullanıcı id'sidir. Bu DLG'a özgü oluşturulan id'dir. Müşteri numarası veya sicil numarası değildir.
message_idStringİletilen mesajın id'sidir. Eğer request içerisinde siz bir messageId vermişseniz bu id bu alanda size geri iletilir. Eğer bir id vermemişseniz DLG sisteminde oluşturulan tekil id bu alandan döner.

Örnek response

{
"recipient_id": "2074022437",
"message_id": "-7523925115459309081"
}
