Metin Mesajı

Kullanıcılarınıza metin mesajlar gönderebilirsiniz.

#İstek URI'ı

https://<DIYALOG-SERVER-API-ENDPOINT>/v1/bots/sendmessage/<YOUR-BOT-ACCESS-TOKEN>

#Örnek İstek

curl -X POST -H "Content-Type: application/json" -d '{
    "recipient": {
     "appCustomerId": "1231232"
    },
    "message": {
        "messageId": "messageId-1",
        "text" : "Hello world !",
        "quickReplies" : [],
        "quickRepliesV2" : []
    }
}' "https://<DIYALOG-SERVER-API-ENDPOINT>/v1/bots/sendmessage/<YOUR-BOT-ACCESS-TOKEN>"

#İstek Json Objesi Detayları

#recipient

receipient objesi içerisinde mesajı hangi kullanıcınıza göndermek istiyorsanız belirtebilirsiniz. Bir müşterinize göndermek istiyorsanız api'ye appCustomerId alanı ile müşteri numarasını iletmeli, bir çalışanınıza göndermek istiyorsanız appStaffId alanı ile sicil numarasını iletmelisiniz.

Eğer mesajı göndermek istediğiniz müşteri veya çalışanın DLG id'sini biliyorsanız, id alanını kullanabilirsiniz.

AlanTipAçıklamaZorunlu Alan
appCustomerIdStringMüşteri numarasıdır.Opsiyonel
appStaffIdStringÇalışan sicil numarasıdır.Opsiyonel
idStringDLG tekil kullanıcı numarasıdır.Opsiyonel

#message

AlanTipAçıklamaZorunlu Alan
messageIdStringSisteminizde oluşturduğunuz tekil mesaj id'sidir. Tüm mesaj iletimleri ve cevapları vs. gibi eventleri DLG bu id ile size besler. Eğer id vermezseniz DLG sizin için bir id üretecek ve bu id üzerinden eventleri iletecektir.Opsiyonel
textStringKullanıcınıza iletmek istediğiniz metindir.Evet
quickRepliesArray<quickReply>Bu alan metin mesajlarda kullanılmamaktadır. Boş array yani "quickReplies" : [] set edilmelidir.Evet
quickRepliesV2Array<quickReplyV2>Bu alan metin mesajlarda kullanılmamaktadır. Boş array yani "quickRepliesV2" : [] set edilmelidir.Evet

#Dönen Yanıt (Response)

Başarılı sonuçlanan API istekleri http-2XX cevap kodu ile json objesini döner. Bu cevap içerisinde mesajın iletildiği kullanıcının DLG tekil id'si ve mesaj id'si döner. Eğer istek içerisinde bir mesaj id verilmiş ise aynı mesaj id size geri dönecek eğer verilmemiş ise DLG tekil bir mesaj id oluşturup size iletecektir.

#Yanıt Json Objesi Detayları

PropertyTipAçıklama
recipient_idStringTekil DLG kullanıcı id'sidir. Bu DLG'a özgü oluşturulan id'dir. Müşteri numarası veya sicil numarası değildir.
message_idStringİletilen mesajın id'sidir. Eğer request içerisinde siz bir messageId vermişseniz bu id bu alanda size geri iletilir. Eğer bir id vermemişseniz DLG sisteminde oluşturulan tekil id bu alandan döner.

Örnek response

{
   "recipient_id": "2074022437",
   "message_id": "-7523925115459309081"
}
