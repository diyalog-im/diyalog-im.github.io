SendOTP API

Kullanıcılarınıza tek seferlik şifre (mobil onay kodu) göndermek isterseniz bu api'yi kullanabilirsiniz. Normal şartlarda SendMessage api'sini de kullanarak kullanıcılarınıza OTP gönderebilirsiniz ama bu api'yi kullanırsanız, gönderdiğiniz otp mesajları ayrı veri tabanlarında istatistiksel olarak tutulmakta ve admin kullanıcılar tarafından istenildiğinde kontrol edilebilmektedir.

SendMessage api'si ile gönderilmesinden bir diğer farkıda DLG Mobil SDK'lar bu otp mesajlarından gelen tek seferlik şifre bilgisini sdk'yı kullanan ana uygulamaya event olarak da haberdar edebilmekte ve bu sayede ana uygulamada otp değerleri otomatik olarak algılanıp doğruluğu kontrol edildikten sonra kullanıcının elle müdahale yapmasına gerek kalmadan işleme devam edebilmesini sağlayabilmektedir.

OTP mesajlarıda metin, görsel ve butonlardan olauşabilir. İsterseniz sadece metin, veya metin ve görsel veya metin, düğmeler veya metin, görsel ve düğmelerden oluşan şekilde otp mesajı iletebilirsiniz.

İstek URI'ı#

https://<DIYALOG-SERVER-API-ENDPOINT>/v1/bots/sendmok/<YOUR-BOT-ACCESS-TOKEN>

Örnek İstek#

curl -X POST -H "Content-Type: application/json" -d '{
"customerNo": "123456789",
"messageId": "messageId-74",
"mokMessage": "Your one time password is \"123456\" ",
"mokValue": "33",
"imageUrl" : "https://dlgmessenger.com/otp.jpg",
"txCode": "LOG2",
"deviceId": "628FAF5A-B690-4E1A-A0EC-1518A6334F0B",
"buttonsV2" :
[
{
"type" :"postback",
"title" : "Approve",
"payload" : "123456",
"payloadHidden" : true,
"payloadSeenByUser": "Onaylıyorum"
},
{
"type" :"postback",
"title" : "Dismiss",
"payload" : "cancelled",
"payloadSeenByUser": "İptal Edildi"
}
]
}' "https://<DIYALOG-SERVER-API-ENDPOINT>/v1/bots/sendmok/<YOUR-BOT-ACCESS-TOKEN>"

İstek Json Objesi Detayları#

AlanTipAçıklamaZorunlu Alan
customerNoStringOtp mesajı göndermek istediğiniz kullanıcının müşteri numarasıdır.Evet
messageIdStringSisteminizde oluşturduğunuz tekil mesaj id'sidir. Tüm mesaj iletimleri ve cevapları vs. gibi eventleri DLG bu id ile size besler. Eğer id vermezseniz DLG sizin için bir id üretecek ve bu id üzerinden eventleri iletecektir.Evet
mokMessageStringMüşterinize göstermek istediğiniz otp mesajıdır.Evet
mokValueStringOtp değeridir. Bu alanı api'ye iletirseniz Mobil DLG SDK otp değerini ana uygulamaya event olarak da gönderir. Bu sayede isterseniz otomatik otp giriş işlemleri yapabilirsiniz.Opsiyonel
imageUrlStringEğer otp mesajınıza görsel eklemek istiyorsanız bu alan görselinizin uri adresini verebilirsiniz.Opsiyonel
txCodeStringLoglama için isterseniz bu alana gönderdiğiniz otp'nin hangi işlem ile ilgili olduğunu yazabilirsiniz. Bu alan sadece geriye dönük incelemelerde kullanılır.Opsiyonel
deviceIdStringLoglama için isterseniz otp mesajını gönderdiğiniz kullanıcının cihaz id'sini bu alandan yazabilirsiniz. Bu alan sadece geriye dönük incelemelerde kullanılır.Opsiyonel
buttonsV2Array<buttonV2>Otp mesajınıza onayla, iptal gibi düğmeler koyabilirsiniz. Bu düğmeler sayesinde kullanıcıya otomatik olarak otp'yi onaylatabilirsiniz.Opsiyonel

buttonV2#

AlanTipAçıklamaZorunlu Alan
typeStringButona basıldığında butonun davranışını belirler. "postback" ve "web_url" olarak iki türü vardır.Evet
titleStringMüşteriye butonların üzerinde gösterilecek metindir.Evet
payloadStringBu alanda verdiğiniz data eğer buton tipi postback ise butona basıldığında size geri iletilir. Bu sayede müşterinin hangi butona bastığını anlayabilirsiniz. Eğer buton tipiniz "web_url" ise bu alanda kullanıcınızn açmasını istediğiniz uri adresini vermelisiniz. Kullanıcı düğmeye bastığında bu uri cihazda açılacaktır.Evet
payloadHiddenBooleanBu flag postback tipi düğmeler için geçerlidir. Gerçekten mesajı ilettiğiniz müşterinin cihazından, payload geldiğinden emin olmak istiyorsanız bu flag'i true set etmeniz gerekiyor. Bu flag true ise mobil cihazda bulunan uygulama anahtarları ile payload imzalanacak ve sizin uygulamanıza iletilecektir. Tabi imzalanmış datayı çözebilmek için sizinde uygulama anahtarlarını biliyor olmanız gerekiyor.Optional
payloadSeenByUserStringBu alan eğer müşteriye butona bastığında chat ekranında payload alanındaki değerden farklı bir mesaj göstermek isterseniz kullanacağınız alandır. Bu alanda verdiğiniz metin kullanıcı düğmeye bastığında chat ekranında görünür.Optional
Önemli Not :

Kullanıcı mobil cihazından, gönderdiğiniz buton objesi olan bir mesajın butonuna bastığında, webhook yapısı üzerinden kullanıcının seçtiği butonun payload alanında verdiğiniz değer uygulamanıza iletilir.

Eğer payloadHidden değerini true yapmışsanız payload mobil istemcide kurumun kendi bildiği DLG platformunun bilmediği anahtarlar ile imzalanıp size o şekilde iletilir. Bu sayede uygulamanız, gerçekten doğru mobil cihazdan buton seçiminin yapılmış olduğundan emin olabilir. Bunun için DLGSDK'nın çalıştığı mobil uygulamanızda imzalama işleminin yapılmış olması gerekmektedir. Ayrıca sizin uygulamanızında aynı anahtarlara sahip olması ve imzalanmış payload'u deşifre edebiliyor olmanız gerekmektedir.

Bir diğer önemli konuda payload objesi için verdiğiniz messageId dir. Düğmeye basıldığında bu alanın değeri payload ile birlikte aşağıdaki formatta uygulamanıza beslenir. Yani istek içerisinde api'ye iletilen mesageId köşeli parantez içerisinde, payload'unuzla ":" karakteri ile ayrılarak iletilir.

Örneğin :
"[messageId-74]:123456"

Eğer payloadHidden true olarak set edilmiş olsaydı bu değer aşağıdaki gibi olacaktır. Yani payload değeri imzalanmış olarak iletilecektir.

"[messageId-74]:12a22113123131213aaab444cb3c224bcb"

Dönen Yanıt (Response)#

Başarılı sonuçlanan API istekleri http-2XX cevap kodu ile cevap json objesini döner.

Yanıt Örneği#

{
"recipient_id": "2074022437",
"message_id": "-7523925115459309081"
}
